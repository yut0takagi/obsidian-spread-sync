あなたのスプレッドシートを、
Obsidian のノートの中へ。 — Spread Sync は、Markdown と Google Sheets を構文ベースでつなぐ。

セルを文中に埋め込む。範囲を表として表示する。表をその場で編集し、ボタン一つでシートへ反映する。 Drive の modifiedTime によるコンフリクト検出、PKCE OAuth、safeStorage 暗号化。

SECTION 01 — 機能

何ができるか

読み取り、表示、編集、書き戻し。すべて Vault の中で完結する。シートが真実、ノートはそのレンズ。

01

インラインセル読み取り

文中に `spread_sync("id", "Sheet1", "B3")` と書けば、その値が Reading View で展開される。Dataview に似た構文。

02

範囲をテーブルで表示

コードブロック ```spread-sync``` で範囲を Markdown テーブルとして描画。ヘッダ・転置・名前付き範囲に対応。

03

その場で編集

editable: true を追加すれば、セルが編集可能になる。変更したセルがハイライトされ、Push ボタンが一括反映する。

04

明示的な書き込み

```spread-write``` ブロックとコマンド Push this block で、意図した書き込みだけを安全に送信する。

05

コンフリクト検出

書き込み前に Drive の modifiedTime を必ず確認。シートが外で変更されていた場合は上書き / キャンセルの選択を求める。

06

PKCE OAuth

ユーザーが自分の GCP プロジェクトで作った Desktop OAuth クライアントを使う。refresh token は Electron safeStorage で暗号化保存。

07

賢いキャッシュ

LRU + debounced persist、Stale-while-revalidate、手動 refresh コマンド。編集中は自動で再取得を停止し、編集内容を保護する。

08

堅牢なエラー処理

429 の指数バックオフ、オフライン検知、用途別エラーバッジ（no access / Excel format / re-auth …）。原因が一目でわかる。

SECTION 02 — 導入

インストール

現時点では手動インストール。Community Plugins への登録は今後対応予定。

# Vault のプラグインディレクトリへ clone
cd /path/to/your/vault/.obsidian/plugins
git clone https://github.com/yut0takagi/obsidian-spread-sync.git spread-sync
cd spread-sync
npm install
npm run build

Obsidian で Settings → Community plugins → Spread Sync を有効化。

SECTION 03 — セットアップ

5 分でつなぐ

Spread Sync はあなた自身の Google Cloud OAuth クライアントを使う。スコープと API クォータをあなたが完全に管理できる。

  1. Google Cloud プロジェクトを作成

    Google Cloud Console を開き、新規プロジェクト作成、または既存プロジェクトを選択。

  2. API を有効化

    APIs & Services → Library で以下を有効化：Google Sheets API ／ Google Drive API。

  3. OAuth consent screen を設定

    User type は External。スコープに .../auth/spreadsheets と .../auth/drive.metadata.readonly を追加。Publishing status は In production にしておく（"Testing" のままだと refresh token が 7日 で失効する）。

  4. OAuth クライアントを作成

    Credentials → Create credentials → OAuth client ID → Application type は Desktop app。Client ID と Client secret の両方をコピー。

  5. Obsidian に入力

    Settings → Spread Sync → OAuth credentials セクションへペースト。Sign in with Google ボタンを押下。ブラウザで認可 → 完了。

クライアントシークレットについて。 Google の公式ドキュメントは、Desktop OAuth の client_secret は 機密保持できない と明言している（あらゆるデスクトップアプリのバイナリ内に同梱されているため）。ローカルの Obsidian 設定に保存することは、公式ライブラリの扱いと完全に一致する。
SECTION 04 — 使い方

構文と例

単一値はインライン記法で、範囲はコードブロックで。編集はオプトイン（editable: true）。書き込みは明示ブロックでのみ。

インラインセル

single cell

文中に値を一つ埋め込む。

今月の店舗数は `spread_sync("<spreadsheet-id>", "Sheet1", "B3")` 店です。

範囲をテーブルに

range

Markdown テーブルとしてレンダリング。ヘッダ・転置のオプションあり。

```spread-sync
id: <spreadsheet-id>
sheet: Sheet1
range: A1:D10
header: true        # 既定 true
transpose: false    # 既定 false
```

編集可能なテーブル

editable ✦

editable: true を追加するとセルをクリックして編集可能。変更したセルが黄色くハイライトされ、テーブル直下の Push ボタンが values:batchUpdate で全変更を一度に反映する。

```spread-sync
id: <spreadsheet-id>
sheet: Sheet1
range: A1:D10
editable: true
```

シート全体

whole sheet

range も named も省略するとシート全体を取得。

```spread-sync
id: <spreadsheet-id>
sheet: Sheet1
```

名前付き範囲

named range

スプレッドシート側で定義した名前付き範囲を参照。シートで範囲を広げても MD は触らなくてよい。

```spread-sync
id: <spreadsheet-id>
sheet: Sheet1
named: MonthlyKPI
```

エイリアス

alias

設定で kpi → <long-spreadsheet-id> と定義しておくと、ノートからは "@kpi" で参照できる。シートを移動しても grep 置換不要。

```spread-sync
id: "@kpi"
sheet: Sheet1
range: A1:C5
```

書き込みブロック

write

明示的なワンショット書き込み。ブロック内にカーソルを置き、Command Palette から Spread Sync: Push this block。

```spread-write
id: <spreadsheet-id>
sheet: Sheet1
target: B3
value: 42
mode: replace        # または "append"
```

範囲書き込みは value を YAML 配列で：

target: A1:B2
value:
  - [1, 2]
  - [3, 4]
SECTION 05 — コマンド

コマンド一覧

すべて Command Palette（Cmd / Ctrl + P）から呼び出せる。

コマンド動作
Refresh current file開いているファイルのキャッシュを無効化し、強制的に再取得する
Refresh all open filesすべての開いている Markdown ファイルに対して同じことを行う
Refresh by spreadsheetid / URL / @alias を入力し、そのスプレッドシート配下のキャッシュをまとめて無効化する
Push this blockカーソル位置の spread-write ブロックをシートへ送信する
SECTION 06 — バッジ

エラーバッジ

何かが失敗したとき、原因を一目で示すバッジとツールチップを表示する。

バッジ意味と対処
⚠ re-authトークン期限切れ／未サインイン。クリックで再認証フローへ。
⚠ no access403 / 404。シートの共有設定または API の有効化を確認。
⚠ Excel format対象が .xlsx として Drive に保存されている。「ファイル → Google スプレッドシートとして保存」で変換する。
⚠ stale 12m直近の取得に失敗、12 分前のキャッシュを表示中。
⚠ offlineネットワーク未接続。
