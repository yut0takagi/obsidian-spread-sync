What it does

Read, render, edit, and push back. All inside your vault. The sheet is the source of truth — your note is the lens.

🔢

Inline cell read

`spread_sync("id", "Sheet1", "B3")` renders as the live cell value. Dataview-style hybrid.

📋

Range as table

Fenced ```spread-sync``` block renders a range as a Markdown table — headers, transpose, named ranges supported.

✏️

Edit in place

Add editable: true — cells become editable. Dirty cells highlight. One click pushes them all back via values:batchUpdate.

🛡️

Conflict-aware

Drive modifiedTime check before every push. If the sheet changed externally, you get an Overwrite / Cancel prompt.

🔐

PKCE OAuth

Your own Google Cloud OAuth client. Refresh token encrypted via Electron safeStorage. Production-published consent screen avoids 7-day expiry.

⚡

Smart caching

LRU + debounced persist. Stale-while-revalidate on file open. Manual refresh commands. Auto-disabled when editing.

Install

Manual install for now — Community Plugins listing is in progress.

# clone into your vault's plugins directory
cd /path/to/your/vault/.obsidian/plugins
git clone https://github.com/yut0takagi/obsidian-spread-sync.git spread-sync
cd spread-sync
npm install
npm run build

Then in Obsidian: Settings → Community plugins → enable Spread Sync.

5-minute setup

Spread Sync uses your own Google Cloud OAuth credentials. You stay in control of scopes and quota.

  1. Create a Google Cloud project

    Open Google Cloud Console. Pick a project or create one.

  2. Enable the APIs

    APIs & Services → Library → enable Google Sheets API and Google Drive API.

  3. Configure OAuth consent screen

    User type: External. Add scopes .../auth/spreadsheets and .../auth/drive.metadata.readonly. Publishing status: In production (avoids the 7-day refresh-token expiry).

  4. Create OAuth client

    Credentials → Create credentials → OAuth client ID → Application type: Desktop app. Copy both the Client ID and Client secret.

  5. Enter in Obsidian

    Settings → Spread Sync → OAuth credentials section → paste both values. Click Sign in with Google. Done.

💡 About the “client secret”: Google's docs explicitly say Desktop OAuth client secrets are not actually confidential — they ship in the binary of every desktop app. Storing it in your local Obsidian settings is consistent with how the official Google libraries handle it.

Usage

Inline for single values, fenced blocks for ranges. Both work in Reading view. Editing requires the editable: true opt-in.

Inline cell

Single value, dropped mid-sentence.

The current store count is `spread_sync("<spreadsheet-id>", "Sheet1", "B3")`.

Range as a table

Renders as a Markdown table.

```spread-sync
id: <spreadsheet-id>
sheet: Sheet1
range: A1:D10
header: true        # optional, default true
transpose: false    # optional, default false
```

✨ Editable table

Add editable: true. Click cells to edit. Push button below the table batches all dirty cells.

```spread-sync
id: <spreadsheet-id>
sheet: Sheet1
range: A1:D10
editable: true
```

Whole sheet

Omit range and named.

```spread-sync
id: <spreadsheet-id>
sheet: Sheet1
```

Named range

```spread-sync
id: <spreadsheet-id>
sheet: Sheet1
named: MonthlyKPI
```

Alias

Define kpi → <long-spreadsheet-id> in settings.

```spread-sync
id: "@kpi"
sheet: Sheet1
range: A1:C5
```

Explicit write block

One-shot push. Place cursor inside the block → Command Palette → Spread Sync: Push this block.

```spread-write
id: <spreadsheet-id>
sheet: Sheet1
target: B3
value: 42
mode: replace      # or "append"
```

Commands

All available from the command palette (Cmd/Ctrl + P).

CommandWhat it does
Refresh current fileInvalidate cache for the active file, force re-fetch
Refresh all open filesSame, for every open markdown leaf
Refresh by spreadsheetPrompt for id/URL/@alias, invalidate all its cache entries
Push this blockSend the current spread-write block to Sheets

Architecture

Four isolated layers. Each one is small, testable, and replaceable.

┌──────────────────────────────────────────────┐
│  Renderer   inline / block postprocessors    │
│              editable table widget           │
├──────────────────────────────────────────────┤
│  Auth       OAuth (PKCE) · safeStorage       │
│              localhost loopback callback     │
├──────────────────────────────────────────────┤
│  Sheets+Drive   pure HTTP (https.request)    │
│                 read · batchRead · write     │
│                 batchWrite · modifiedTime    │
│                 429 backoff                  │
├──────────────────────────────────────────────┤
│  Storage    LRU cache · debounced persist    │
└──────────────────────────────────────────────┘
